 # 3. Git & GitHub
 
 - tools for managing changes to (text) file  contents 
 - uses repositories (хранилище) to store and track files
 - git is distributed source-control system

## Base Terminology
 - repository (repo) - a directory that contains files and folders
 - staging area - a temporary area where changes are stored before committing
 - .gitignore - a file that specifies which files and directories to ignore in a repository
 - .git - a hidden directory that contains all the information about the repository
 - commit - a snapshot of the repository at a specific point in time
 - branch - a separate line of development in a repository
 - merge - combining changes from one branch into another
 - clone - creating a copy of a repository
 - push - uploading local changes to a remote repository
 - pull - downloading changes from a remote repository to local machine

## Git Commands
- git init - initialize a new Git repository
- git clone <repository> - create a copy of a remote repository
- git status - check the status of the repository
  - if new file is created, it must be added to the staging area before committing
  - if file is modified, it must be added to the staging area before committing
- git add <file> - add a file to the staging area
- git add . - add all files to the staging area
- git commit -m "message" - commit changes to the repository with a message
- git log - view the commit history
- git diff - view the differences between the working directory and the staging area
- git remote
  - git remote add <name> <repository> - add a remote repository
  - git remote remove <name> - remove a remote repository
  - git remote -v - view the remote repositories
- git push <remote> <branch> - upload local changes to a remote repository
- git pull <remote> <branch> - download changes from a remote repository
